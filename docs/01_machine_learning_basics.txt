CHAPTER 1: INTRODUCTION TO MACHINE LEARNING
===========================================

What is Machine Learning?
--------------------------
Machine Learning (ML) is a type of artificial intelligence that allows computers to learn from data without being explicitly programmed for every task. Instead of writing rules manually, we feed the computer lots of examples and it figures out the rules on its own.

Think of it this way: instead of telling a computer "if the email contains the word FREE and has lots of exclamation marks, it is spam", we show it thousands of spam emails and thousands of normal emails, and it learns to tell the difference by itself.

The Three Types of Machine Learning
-------------------------------------

1. SUPERVISED LEARNING
Supervised learning is when we train a model using labeled data. "Labeled" means every example in our training data already has the correct answer attached to it.

Example: We have 10,000 photos of cats and dogs. Each photo is labeled "cat" or "dog". We train a model on these photos. After training, the model can look at a new photo and predict whether it shows a cat or a dog.

Common supervised learning tasks:
- Classification: predicting a category (spam/not spam, cat/dog, disease/no disease)
- Regression: predicting a number (house price, temperature tomorrow, stock value)

Popular supervised learning algorithms:
- Linear Regression (for predicting numbers)
- Logistic Regression (for binary classification)
- Decision Trees
- Random Forests
- Support Vector Machines (SVM)
- Neural Networks

2. UNSUPERVISED LEARNING
Unsupervised learning is when we train a model on data that has NO labels. The model must find hidden patterns or structure in the data on its own.

Example: A retailer has purchase history for 1 million customers but no labels. Using unsupervised learning, the model might discover that customers naturally cluster into groups: "budget shoppers", "luxury buyers", "seasonal buyers", etc. The model found these groups without anyone telling it they existed.

Common unsupervised learning tasks:
- Clustering: grouping similar data points together (K-Means, DBSCAN)
- Dimensionality Reduction: simplifying data while keeping important information (PCA, t-SNE)
- Anomaly Detection: finding unusual data points

3. REINFORCEMENT LEARNING
Reinforcement learning is when an agent learns by taking actions in an environment and receiving rewards or penalties. There are no labeled examples — the agent learns purely through trial and error.

Example: Training a computer to play chess. The agent makes moves, and if it wins it gets a positive reward. If it loses, it gets a negative reward. Over millions of games it learns which moves lead to winning.

Famous example: AlphaGo by DeepMind used reinforcement learning to become the world's best Go player, defeating human world champions in 2016.

Key Machine Learning Concepts
-------------------------------

TRAINING DATA vs TEST DATA
Before training a model, we always split our dataset into two parts:
- Training set (usually 80% of data): used to train the model
- Test set (usually 20% of data): used to evaluate how well the model performs on new, unseen data

This split is important because we want to know if our model has truly learned general patterns, not just memorized the training examples.

OVERFITTING
Overfitting happens when a model learns the training data TOO well — including the noise and random variations — and then performs poorly on new data.

Analogy: Imagine a student who memorizes the exact answers to last year's exam questions but doesn't understand the underlying concepts. They will fail if the exam questions are different.

Signs of overfitting:
- Very high accuracy on training data
- Much lower accuracy on test data

Solutions to overfitting:
- Use more training data
- Use regularization techniques (L1, L2)
- Use simpler models
- Use dropout (for neural networks)

UNDERFITTING
Underfitting is the opposite — the model is too simple and hasn't learned enough from the training data. It performs poorly on both training and test data.

Analogy: A student who barely studied and can't even answer the basic questions they've seen before.

THE BIAS-VARIANCE TRADEOFF
This is one of the most fundamental concepts in machine learning:
- High bias = underfitting (model is too simple, makes strong assumptions)
- High variance = overfitting (model is too complex, memorizes training data)

The goal is to find the sweet spot with low bias AND low variance.

FEATURES
Features are the input variables used to make predictions. Choosing the right features is called "feature engineering" and is one of the most important skills in machine learning.

Example: To predict house prices, your features might be:
- Square footage
- Number of bedrooms
- Number of bathrooms
- Neighborhood
- Year built
- Distance to city center

LABELS / TARGET VARIABLE
The label is what you are trying to predict. In the house price example, the label is the price itself.

MODEL EVALUATION METRICS
How do we know if a model is good? We use metrics:

For classification:
- Accuracy: percentage of correct predictions
- Precision: of all the times we predicted "yes", how often were we right?
- Recall: of all the actual "yes" cases, how many did we catch?
- F1 Score: harmonic mean of precision and recall
- AUC-ROC: measures ability to distinguish between classes

For regression:
- Mean Absolute Error (MAE): average of absolute differences between predictions and actual values
- Mean Squared Error (MSE): average of squared differences (penalizes large errors more)
- Root Mean Squared Error (RMSE): square root of MSE, in same units as target
- R-squared: proportion of variance explained by the model (0 to 1, higher is better)

The Machine Learning Pipeline
-------------------------------
A typical ML project follows these steps:

Step 1: Define the problem
- What are you trying to predict?
- What kind of ML task is this (classification, regression, clustering)?
- What does success look like?

Step 2: Collect data
- Gather relevant data from databases, APIs, web scraping, surveys, etc.
- More data is almost always better

Step 3: Explore the data (EDA - Exploratory Data Analysis)
- Understand the distribution of your data
- Find missing values, outliers, and anomalies
- Visualize relationships between features

Step 4: Prepare the data
- Handle missing values (impute or remove)
- Encode categorical variables (one-hot encoding, label encoding)
- Scale/normalize numerical features
- Split into training and test sets

Step 5: Choose and train a model
- Select appropriate algorithm(s)
- Train on the training data
- Tune hyperparameters using cross-validation

Step 6: Evaluate the model
- Test on the held-out test set
- Calculate relevant metrics
- Compare to a baseline (e.g., random guessing)

Step 7: Deploy the model
- Integrate into a product or system
- Monitor performance over time
- Retrain when performance degrades

Popular Machine Learning Libraries
------------------------------------
- Scikit-learn: the most popular ML library for Python. Implements almost every classical ML algorithm.
- Pandas: for data manipulation and analysis
- NumPy: for numerical computing
- Matplotlib / Seaborn: for data visualization
- XGBoost / LightGBM: powerful gradient boosting libraries, often win Kaggle competitions

Real-World Applications of Machine Learning
---------------------------------------------
- Email spam filtering (classification)
- Netflix/Spotify recommendations (collaborative filtering)
- Fraud detection in banking (anomaly detection)
- Medical diagnosis from X-rays (image classification)
- Self-driving cars (reinforcement learning + computer vision)
- Google Translate (sequence-to-sequence models)
- Voice assistants like Siri and Alexa (speech recognition + NLP)
- Stock price prediction (regression, though very difficult)
- Face unlock on phones (face recognition)
- Weather forecasting (regression/time series)
